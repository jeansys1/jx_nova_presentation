Why JX Nova

The problem we solve
Learning alone is hard — motivation drops fast without accountability
Tutorials don't replace real hands-on project experience
The job market expects a portfolio and collaborative experience
Junior developers often lack accessible feedback and mentorship
What JX Nova brings
A structured weekly peer-learning group with a clear agenda
Shared GitHub projects to build a real, public portfolio
Active preparation for interviews, internships and freelance work
Resource sharing, cross code reviews and measurable progress

Stack & organisation

Discord
Daily communication, topic channels, voice calls
GitHub
Code, shared projects, PR reviews, public portfolio
Notion
Roadmap, resources, meeting notes (phase 2)
How we work
1 weekly meeting of 20–30 min with agenda sent 24h in advance
1 active shared project on GitHub at all times
Mandatory cross code review before any merge
Each member shares at least one resource per week
Monthly retrospective: what works, what we improve

Launch roadmap

1
Week 1 — Foundations
Set up structured Discord, GitHub org JX-Nova, presentation README. Write the recruitment pitch and define member selection criteria.
2
Weeks 2–3 — Recruitment
Share in dev communities (Discord, Reddit, school). Informal 10-min interviews. Select 2 to 4 motivated members available at least 4h/week.
3
Month 1 — First shared project
Concrete JS mini-project (web app, API, CLI tool). Every member contributes to the same repo. Delivered in 4 weeks and published publicly.
4
Month 2+ — Routine & job market
Progressively more complex projects. Build individual portfolios. Apply for internships and freelance with the group as a reference. Open to new members when ready.
